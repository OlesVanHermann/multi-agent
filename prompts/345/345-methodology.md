# 345 — Methodology

## Étapes
1. Lire la spec dans project/pipeline/342-output/
2. Lire le contexte dans memory.md
3. Pour chaque exigence fonctionnelle :
   - Créer la fonction/classe correspondante
   - Ajouter docstring avec référence à l'EF
   - Gérer les edge cases
4. Assembler le code :
   - Imports en haut
   - Classes/fonctions
   - Point d'entrée main
5. Vérifier que le code s'exécute sans erreur
6. Écrire dans project/pipeline/345-output/

## Règles
- Chaque fonction référence l'exigence qu'elle implémente
- Nommage explicite (pas de variables a, b, c)
- Gestion d'erreurs pour les I/O
- PEP 8 respecté
