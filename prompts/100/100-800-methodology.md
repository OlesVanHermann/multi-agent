# 100-800 — Methodology

## Étapes
1. Lire le dernier bilan dans `bilans/100-cycle*.md`
2. Lire la methodology actuelle de 100
3. Pour chaque problème :
   - Formuler une règle corrective concrète
   - Exemple : si "agent bloqué non détecté" → ajouter "checker status toutes les 2 min"
4. Intégrer les nouvelles règles
5. Conserver les règles qui fonctionnent (score > 80%)
6. Écrire dans `prompts/100/100-methodology.md`

## Principes
- Une règle = un problème
- Préférer les exemples concrets
- Si problème récurrent 2 cycles → escalader à 100-900
