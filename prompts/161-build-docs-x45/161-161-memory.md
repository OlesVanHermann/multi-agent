## Auth freemium (reference : ~/aiapp/backend/AUTH.md)

### Architecture auth
- nginx :80/:443 (TLS termination, security headers)
- `AuthEnforcementMiddleware` dans `server_common.py` : tout `/api/*` requiert Bearer JWT
- Path normalization (posixpath.normpath + unquote) anti-traversal
- Keycloak :8040 realm `freemium` (JWT issuer, RS256)
- `Depends(get_current_user)` dans `keycloak_auth_api.py` pour identite user

### Flow JWT
1. Decode JWE phantom token (local, rapide)
2. Sinon verify pyjwt RS256 via JWKS Keycloak
3. Issuer verification (realm freemium)
4. Cache Redis 30s TTL
5. Return user dict (sub, email, preferred_username, roles) ou 401

### Public paths (PAS de JWT requis)
- `/api/auth/` — login flow Keycloak
- `/api/health` — healthchecks
- `/api/search/` — auth optionnelle (`_get_optional_user`)
- `/api/transfer/download/` — token 43 chars
- `/api/visio/webhooks/` — callbacks externes

### Storage auth
- PostgreSQL :5432 — user `freemium`, per-user DBs `db_{keycloak_uuid}`
- Redis :6380 — requirepass
- MinIO :9000 — access-key/secret-key, buckets: `drive`, `stalwart-archive`, `mail-{username}`

### Mail auth (Stalwart)
- IMAP :993 XOAUTH2 via Keycloak JWT
- SMTP :25 no auth (MX receiver, STARTTLS)
- JMAP :10443 Bearer token

- **TOUJOURS** tester `test_*_requires_auth` → 401 sans token
- Admin endpoints : verifier role admin dans user dict
- Prefix : `/api/{service}/{resource}` (jamais sans `/api/`)
- Frontend : `Authorization: Bearer <token>` sur chaque fetch
