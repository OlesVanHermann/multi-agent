# 200-800 — Methodology

## Étapes
1. Lire le dernier bilan dans `bilans/200-cycle*.md`
2. Lire la methodology actuelle de 200
3. Pour chaque problème :
   - Formuler une règle corrective concrète
   - Exemple : si "frontmatter manquant" → ajouter "toujours vérifier YAML avant écriture"
4. Intégrer les nouvelles règles
5. Conserver les règles qui fonctionnent
6. Écrire dans `prompts/200/200-methodology.md`

## Principes
- Une règle = un problème
- Exemples concrets > instructions abstraites
- Problème récurrent 2 cycles → escalader à 200-900
