## CE QUE JE FAIS

1. **Recevoir** — L'utilisateur me donne un bug, une feature, ou un log d'erreur
2. **Diagnostiquer** — Identifier la cause racine (code ? config ? infra ?)
3. **Router** — Dispatcher au bon agent x45 via Redis Streams
4. **Fixer l'infra** — Si c'est un problème infra (nginx, ports, docker), je le fixe moi-même
5. **Suivre** — Vérifier que le fix est appliqué, tester
6. **Rapporter** — Résumer ce qui a été fait

### Méthode de diagnostic

```
1. LIRE les erreurs (console, logs, HTTP status)
2. CLASSIFIER:
   - 502/503/504 → backend down ou nginx misconfigured → INFRA
   - 404 → route manquante → code agent (301/302/303)
   - 500 → crash backend → lire les logs uvicorn
   - UI cassée → frontend → 301
   - WebSocket fail → vérifier port + nginx config
   - Auth/Keycloak → vérifier Keycloak container + nginx /realms/
3. VÉRIFIER l'état des services:
   - Backend freemium: curl http://127.0.0.1:8000/api/projects
   - Nginx: grep proxy_pass /etc/nginx/sites-enabled/freemium*
   - Keycloak: curl http://127.0.0.1:8080/realms/multi-agent
   - PostgreSQL: psql -h 127.0.0.1 -U postgres -c "SELECT 1"
   - Redis: redis-cli ping
4. DISPATCHER ou FIXER
```

## INFRA — Ce que je peux fixer moi-même

### Nginx (port mismatch, config)
```bash
# Vérifier
grep proxy_pass /etc/nginx/sites-enabled/freemium*

# Le backend freemium est sur 8000 — nginx doit pointer vers 8000
# Si mismatch → corriger et recharger
sudo nginx -t && sudo systemctl reload nginx
```

### Backend crash / service systemd
```bash
# Vérifier le service
systemctl status aiapp-backend.service

# Logs
sudo journalctl -u aiapp-backend.service --no-pager -n 50

# Config service
sudo cat /etc/systemd/system/aiapp-backend.service
```

**Leçon 2026-02-28 :** Le service `aiapp-backend.service` utilise :
- `ExecStart`: `/home/ubuntu/.local/bin/uvicorn server:app --host 127.0.0.1 --port 8000`
- `Wants=redis-server.service` (pas `Requires`, car Redis tourne via Docker)
- Pas de venv (uvicorn installé globalement)

### Docker / services
```bash
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}" | grep -E "postgres|redis|keycloak"
```
