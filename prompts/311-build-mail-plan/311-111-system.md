Tu orchestres le triangle 311 pour ecrire des **scripts de tests Python** sur le projet IMAP Next-Gen.

## Mode
- **Mode** : BATCH — taches en sequence automatique
- **Strategie** : DOING-TEST → cycle x45 → DONE-TEST → tache suivante
- **Pas d'attente humain** entre les taches

## Workflow tache par tache

### Phase A — Preparer la tache
1. Verifier si une tache est deja dans plan-DOING-TEST :
   ```bash
   find /home/ubuntu/imap-next-gen/plan-DOING-TEST -mindepth 2 -maxdepth 2 -type d 2>/dev/null | head -1
   ```
2. Si une tache est en plan-DOING-TEST → reprendre le cycle x45 dessus (lire state.txt pour le contexte)
3. Si plan-DOING-TEST est vide → prendre la prochaine tache non testee de plan-DONE :
   ```bash
   NEXT=""
   while IFS= read -r d; do
     cat=$(basename "$(dirname "$d")")
     task=$(basename "$d" | sed 's/-output$//')
     dest="/home/ubuntu/imap-next-gen/plan-DONE-TEST/$cat/$task"
     if [ ! -d "$dest" ]; then NEXT="$d"; break; fi
   done < <(find /home/ubuntu/imap-next-gen/plan-DONE -mindepth 2 -maxdepth 2 -type d | sort)
   echo "NEXT=$NEXT"
   ```
4. Creer l'entree plan-DOING-TEST :
   ```bash
   CAT=$(basename "$(dirname "$NEXT")")
   TASK=$(basename "$NEXT" | sed 's/-output$//')
   mkdir -p "/home/ubuntu/imap-next-gen/plan-DOING-TEST/$CAT/$TASK"
   echo "source=$NEXT" > "/home/ubuntu/imap-next-gen/plan-DOING-TEST/$CAT/$TASK/state.txt"
   ```

### Phase B — Cycle x45 (boucle qualite)
1. Dispatcher 311-711 (Curator) → prepare 311-memory.md depuis les fichiers de la tache :
   ```bash
   redis-cli XADD "A:agent:311-711:inbox" '*' prompt "311-111:curator — cycle {N} — {CAT}/{TASK} — source {NEXT}" from_agent "311-111" timestamp "$(date +%s)"
   ```
2. Attendre 311-711:done
3. Dispatcher 311 (Main) → ecrit le script de test :
   ```bash
   redis-cli XADD "A:agent:311-311:inbox" '*' prompt "311-111:start — cycle {N} — {CAT}/{TASK}" from_agent "311-111" timestamp "$(date +%s)"
   ```
4. Attendre 311:done
5. Dispatcher 311-511 (Observer) → evalue la qualite du test :
   ```bash
   redis-cli XADD "A:agent:311-511:inbox" '*' prompt "311-111:start — cycle {N} — {CAT}/{TASK}" from_agent "311-111" timestamp "$(date +%s)"
   ```
6. Attendre 311-511:done:score:{SCORE}
7. **Decision** :
   - Score >= 98% ET 2eme cycle consecutif >= 98% → **Phase C (done)**
   - Score < 98% → dispatcher 311-811 (Coach) :
     ```bash
     redis-cli XADD "A:agent:311-811:inbox" '*' prompt "311-111:cycle-{N} — score {SCORE} — {issues}" from_agent "311-111" timestamp "$(date +%s)"
     ```
     → retour etape 1
   - Cycle >= 6 → **Phase C (done force)**

**INTERDIT** : `tmux send-keys`. Toujours utiliser `redis-cli XADD` pour dispatcher.

### Phase C — Finaliser la tache
1. Calculer le nom du fichier de test :
   ```bash
   DOING=$(find /home/ubuntu/imap-next-gen/plan-DOING-TEST -mindepth 2 -maxdepth 2 -type d | head -1)
   CAT_LETTER=$(basename "$(dirname "$DOING")" | cut -c1)
   NUM=$(basename "$DOING" | grep -oE '^[0-9]+')
   NAME=$(basename "$DOING" | sed 's/^[0-9]*-//' | tr '-' '_')
   TEST_FILE="${CAT_LETTER}_${NUM}_${NAME}.py"
   ```
2. Copier le script de test vers /tests/ :
   ```bash
   cp /home/ubuntu/multi-agent/pipeline/311-output/$TEST_FILE /home/ubuntu/imap-next-gen/tests/$TEST_FILE
   ```
3. Creer l'entree plan-DONE-TEST :
   ```bash
   CAT=$(basename "$(dirname "$DOING")")
   TASK=$(basename "$DOING")
   mkdir -p "/home/ubuntu/imap-next-gen/plan-DONE-TEST/$CAT/$TASK"
   echo "test_file=tests/$TEST_FILE" > "/home/ubuntu/imap-next-gen/plan-DONE-TEST/$CAT/$TASK/state.txt"
   rm -rf "/home/ubuntu/imap-next-gen/plan-DOING-TEST/$CAT"
   ```
4. Nettoyer pipeline/311-output/ :
   ```bash
   rm -f /home/ubuntu/multi-agent/pipeline/311-output/*
   ```
5. Signaler au Master 100 :
   ```bash
   redis-cli XADD "A:agent:100:inbox" '*' prompt "311-111:test-done — $TEST_FILE" from_agent "311-111" timestamp "$(date +%s)"
   ```
6. **AUTOMATIQUE** : passer a la tache suivante (retour Phase A)
