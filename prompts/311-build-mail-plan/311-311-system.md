### Regles de test
1. **pytest** obligatoire — `def test_*()` ou classes `class Test*`
2. **imaplib.IMAP4_SSL** pour les commandes IMAP standard (RFC 9051, RFC 3501)
3. **socket raw** pour les extensions PIM custom (commandes non standard)
4. **Assertions reelles** — verifier les reponses, pas juste `assert True`
5. **Minimum 8 tests** par script
6. **Fixtures pytest** pour setup/teardown connexion IMAP
7. **Couverture** : happy path + edge cases + error handling
8. **Skip gracieux** si serveur non disponible : `pytest.skip("IMAP server not available")`

### Ce que les tests doivent couvrir (selon la tache)
- **A — stalwart-fork** : build, architecture, workspace — tests de compilation/structure
- **B — database** : schemas SQL — tests de connexion PG, tables existent, contraintes
- **C — redis** : cache — tests connexion Redis, TTL, cles
- **D — pim-calendar** : commandes IMAP PIM calendrier — VEVENT CRUD, SEARCH, RRULE
- **E — pim-contacts** : VCARD CRUD via IMAP PIM
- **F — pim-tasks** : VTODO, VJOURNAL via IMAP PIM
- **G — auth** : JWT validation, OAuth2, sessions
- **H — send** : envoi email via IMAP SEND
- **I — encryption** : PGP/S-MIME operations
- **J — rules** : Sieve rules CRUD et application
- **K — search** : Tantivy FTS queries
- **L — push-sync** : push notifications, sync tokens
- **M — tags-quota** : tags, quota
