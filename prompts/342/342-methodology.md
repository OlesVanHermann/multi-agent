# 342 — Methodology

## Étapes
1. Lire les rapports de 341 dans project/pipeline/341-output/
2. Lire le contexte dans memory.md
3. Extraire les exigences fonctionnelles des recommandations
4. Identifier les contraintes techniques
5. Définir les critères d'acceptance
6. Assembler la spec :
   # Spécification technique
   ## Contexte
   ## Exigences fonctionnelles
   ## Contraintes techniques
   ## Critères d'acceptance
   ## Dépendances
7. Écrire dans project/pipeline/342-output/

## Règles
- Chaque exigence est numérotée (EF-001, EF-002...)
- Chaque exigence a un critère d'acceptance mesurable
- Les contraintes citent leur source (rapport 341)
- Pas d'ambiguïté : un développeur doit pouvoir coder directement
