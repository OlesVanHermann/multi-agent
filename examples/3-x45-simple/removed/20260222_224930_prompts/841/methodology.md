# 841 — Methodology

## Étapes
1. Lire le dernier bilan de 500 dans `project/bilans/341-cycle*.md`
2. Lire la methodology actuelle de 341
3. Pour chaque problème identifié dans le bilan :
   - Formuler une règle corrective concrète
   - Exemple : si "chiffres non sourcés" → ajouter "citer la ligne du CSV pour chaque chiffre"
4. Intégrer les nouvelles règles dans la methodology existante
5. Conserver les règles qui fonctionnent (score > 80%)
6. Écrire la methodology mise à jour dans `prompts/341/methodology.md`

## Principes
- Une règle = un problème. Pas de règles vagues.
- Préférer les exemples concrets aux instructions abstraites
- Si un problème revient 2 cycles de suite, escalader à 945 via Redis
